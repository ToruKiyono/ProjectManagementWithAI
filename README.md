# 项目管理软件初版原型（同步版）

当前版本已按你的要求调整为：

- 不以手工录入为主
- 数据来源为外部平台接口或 Excel 导入（CSV）
- 每次更新保留“更新前”快照，并提供前后对比
- 支持多版本并行/交叉（如 HC、HCS、HCSO、会战版本）

## 运行方式

1. 直接打开 `index.html`
2. 推荐使用本地静态服务器打开（避免浏览器跨域限制）

## 已实现能力

- 需求流程内置：
  `需求基线 -> 需求澄清 -> 设计文档编写 -> 需求会签 -> 需求评审 -> 需求串讲 -> 需求开发（跟进度） -> 需求转测 -> 问题单清理 -> 需求上线`
- 问题单流程内置：
  `待确定 -> 已接纳 -> 修复中 -> 已修复 -> 已转测 -> 待验收 -> 关闭`
- 数据同步方式：
  - 双接口：一个接口返回需求，一个接口返回问题单
  - 单接口：返回 `{ requirements, issues }`
  - Excel 导入：分别导出“需求 CSV”和“问题单 CSV”后导入
- 前后对比：
  - 新增 / 删除 / 变更
  - 变更字段明细（按 ID 比较）
- 版本维度能力：
  - 需求与问题单均支持 `version` 字段
  - 页面可按版本筛选
  - 时间图按版本分组，各版本使用独立时间轴
- 阶段时间轴：按需求 10 个阶段节点时间绘制时间点（支持并行版本）
- 大数据量展示：需求/问题单/时间轴支持分页（默认每页 50/50/30）
- 风险提示：
  - 当前阶段超期
  - 下一阶段临近/超期
  - 开发进度落后于时间进度
  - 问题单超 SLA / 截止日期超期
  - 上线或问题清理阶段仍有未关闭问题单
- 本地持久化：`localStorage` 保存当前快照和上次快照

## 接口返回建议

### 需求接口（数组或对象包装）

支持以下形式之一：

- 直接数组：`[ {...}, {...} ]`
- 对象包装：`{ requirements: [...] }` 或 `{ data: [...] }`

单条需求推荐字段：

- `id`
- `title`
- `owner`
- `version`
- `stage`
- `progress`
- `startDate`
- `endDate`
- `stageDates`（对象，键为阶段名）

需求也兼容常见中文字段名，例如：

- `编号` -> `id`
- `标题` / `描述` -> `title`
- `发布计划` / `迭代计划` / `归属` / `归属微服务` / `Offering` / `客户` -> `version`
- `责任人` / `创建人` / `测试责任人` -> `owner`
- `状态` / `研发进展` -> `stage`
- `进度百分比` / `进度` -> `progress`
- `开始时间` / `创建时间` -> `startDate`
- `完成时间` / `关闭时间` / `规划上线时间` -> `endDate`

### 问题单接口（数组或对象包装）

支持以下形式之一：

- 直接数组：`[ {...}, {...} ]`
- 对象包装：`{ issues: [...] }` 或 `{ data: [...] }`

单条问题单推荐字段：

- `id`（界面展示为“问题单号”）
- `title`
- `version`
- `requirementId`
- `severity`
- `owner`
- `status`
- `createdAt`
- `dueDate`

问题单也兼容常见中文字段名，例如：

- `问题编号` -> `id`
- `问题标题` / `摘要` / `问题描述` -> `title`
- `发现问题版本` / `发现版本` / `发布计划` / `迭代` -> `version`
- `关联工作项` -> `requirementId`
- `严重程度` / `优先级` -> `severity`
- `当前责任人` / `责任人` / `研发责任人` -> `owner`
- `问题状态` / `挂起/撤销` -> `status`
- `问题创建时间` / `发现时间` -> `createdAt`
- `期望修复时间` / `承诺修复时间` -> `dueDate`

## CSV 字段建议

### 需求 CSV

推荐表头：

`id,title,version,owner,stage,progress,startDate,endDate,需求基线,需求澄清,设计文档编写,需求会签,需求评审,需求串讲,需求开发（跟进度）,需求转测,问题单清理,需求上线`

也支持直接导入中文表头的需求 CSV，例如至少包含：

`编号,标题,发布计划,责任人,状态,进度百分比,开始时间,完成时间`

### 问题单 CSV

推荐表头：

`id,title,version,requirementId,severity,owner,status,createdAt,dueDate`

也支持直接导入中文表头的缺陷 CSV，例如至少包含：

`问题编号,问题标题,发现问题版本,关联工作项,严重程度,当前责任人,问题状态,问题创建时间,期望修复时间`
